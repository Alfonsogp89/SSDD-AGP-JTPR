1. Introducción
Introducción
Estas transparencias describen el proyecto de prácticas de Sistemas
Distribuidos
Constará del desarrollo de una aplicación distribuida formada por
un conjunto de servicios:
1 gestión de usuarios y autenticación (JWT),
2 estadísticas de acceso y monitorización (Prometheus),
3 servicio de sesión de prompt LlamaChat,
4 servicio de log de sesiones de prompt,
5 pruebas End-to-End (E2E) automáticas
Se hará uso de las tecnologías REST, gRPC, Kafka, Docker,
Prometheus y Selenium

2. Especificación del proyecto
Introducción
Objetivo
El objetivo del proyecto es escribir una aplicación distribuida compuesta
de varios contenedores Docker que ofrecen la funcionalidad de gestionar
usuarios, estadísticas de acceso y uso del servicio, sesiones de prompt,
log de sesiones y validación automática del interfaz (E2E).
Los servicios se desarrollarán con REST y con gRPC
Cada servicio se instalará en un contenedor independiente
Se tendrán que diseñar los APIs de los servicios en gRPC y los
servicios REST que corresponda
La seguridad es fundamental: se usará JWT (JSON Web Tokens)
para todas las autenticaciones entre servicios y clientes

Capas de la aplicación
La aplicación se dividirá en las tres capas estándar de las aplicaciones
distribuidas:
Frontend (F) El frontend se ofrecerá a través de un acceso Web. Se
podrá implementar usando tecnologías de cliente como Bootstrap1. Y
la parte servidor, que formará parte del backend , se podrá
implementar haciendo uso de cualquier tecnología, desde PHP hasta
Node.js, pasando por Python Flask (veremos un ejemplo)
Backend (B) El backend incluye: (1) los servidores relativos a la vista; y
(2) los servidores que implementan los servicios REST y gRPC en
Java
Datos (D) La capa de datos utilizará una o varias bases de datos para
almacenar los datos de todos los servicios
1https://getbootstrap.com/.

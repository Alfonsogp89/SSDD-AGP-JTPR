app/app.py
En el directorio app está el fichero app.py
Incluye las definiciones necesarias y la variable app, la aplicación
A partir de ese momento, se pueden ir añadiendo rutas de la aplicación con @app.route(ruta)
En la documentación aparecen las posibilidades, pero las funciones más importantes son:
render_template(): genera un resultado de un template (ejemplo de página con variables) de los que existen en el directorio templates (se verá después)
redirect(): redirecciona a otra página. Se puede usar la función url_for(), que da la URL de una función definida con @app.route
Aunque en el ejemplo no se usa, flask ofrece también el objeto (un diccionario de Python) session
from flask import session

Patrones/Templates
Flask utiliza los templates de Jinja2, otro paquete de Python
https://jinja.palletsprojects.com/en/3.0.x
Los patrones permiten hacer bucles, sustituir valores de variables, ejecutar funciones de Python, heredar de otros patrones, definir bloques, etc.
En el proyecto están en el subdirectorio templates
Se generan con la función render_template()
En el ejemplo hay una plantilla base, base.html, y las demás heredan de ella, redefiniendo los bloques que se definen
La sustitución de variables y llamada a funciones se hace con la construcción doble llaves: {{ }}
Las funciones de iteración, herencia, bloques, etc, se construyen con {% %}:
{% block <nombre> %}
{% endblock %}

Login
El login se consigue con el paquete flask_login
Define ciertas funciones, como login_user, para hacer login de un usuario, @login_required, para poder acceder a una ruta sólo cuando el usuario está autenticado
Se usa el objeto LoginManager, y hay que llamar a init_app con el objeto aplicación
También define un mixin, UserMixin, que hay que mezclar con la definición de nuestro usuario, para mantener los datos del login
Está en el fichero app/models.py, donde se pueden añadir los datos necesarios y veremos cómo conectarlo con la base de datos

Peticiones REST
El frontend tiene que hacer peticiones REST a los servicios internos
Se muestra cómo hacerlas de manera sencilla con Python
Usualmente el paquete requests contiene todo lo necesario para realizarlas
Incluye todos los métodos de petición REST, los códigos de respuesta y también métodos auxiliares .json()

Peticiones REST II
import requests
response = requests.get("url")
# response.text contiene el texto de la respuesta
# response.json convierte el texto a un objeto JSON
# response.status_code contiene el código devuelto

Peticiones REST III
Las peticiones POST contienen información que se envía al servidor para pedir servicios

Peticiones REST IV
userlogin = {"email": "abc@def.com", "password": "blah"}
response = requests.post("backend-server/login_attempt", json=userlogin)

Peticiones REST V
headers = {"Content-Type": "application/json"}
response = requests.post("backend-server/login_attempt", data=json.dumps(userlogin), headers=headers)

JSON Web Tokens (JWT)
En sistemas distribuidos, mantener la sesión en la memoria del servidor es problemático
JWT es un estándar abierto para transmitir información de forma segura entre partes como un objeto JSON
Es stateless: el servidor no guarda nada. Toda la información va en el token

Flujo de autenticación con JWT
1. El usuario envía credenciales
2. El backend valida
3. Si son correctas, genera un JWT
4. El cliente guarda el token
5. En peticiones protegidas, envía:
   Authorization: Bearer <token>
6. El backend verifica la firma

Implementación JWT en Python
pip install pyjwt

Implementación JWT en Python II
SECRET_KEY = "mi_secreto_super_seguro"
payload = {
  'sub': user_id,
  'iat': datetime.datetime.utcnow(),
  'exp': datetime.datetime.utcnow() + datetime.timedelta(minutes=30)
}
token = jwt.encode(payload, SECRET_KEY, algorithm='HS256')

Implementación JWT en Python III
jwt.decode(token, SECRET_KEY, algorithms=['HS256'])

Uso de JWT desde el cliente
token = session.get("jwt_token")
headers = {"Authorization": f"Bearer {token}"}
response = requests.get(url, headers=headers)
